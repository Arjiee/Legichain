# Prompt 1 — Update Main Dashboard with Ordinance Monitoring

**Prompt:**

Update the **Main Dashboard of the GMA Cavite Blockchain Barangay Ordinance Monitoring System** to display real-time ordinance information and violation statistics for each barangay.

### 1. Ordinance Overview Panel

Add a **Main Dashboard section called "Ordinance Monitoring Overview".**

Display three system counters that update automatically based on stored records:

• **Published Documents** – total number of ordinances uploaded in the system
• **Verified on Chain** – total number of ordinance or violation records verified on the blockchain
• **Active Ordinances** – total number of active ordinances across all barangays

These counters must **update dynamically in real time when new records are added or verified**.

---

### 2. Barangay Ordinance Table

Create a **central table on the dashboard that shows all ordinances per barangay in GMA Cavite.**

Table columns:

* Ordinance ID
* Barangay Name
* Ordinance Title
* Category
* Date Implemented
* Violations This Year
* Total Violations
* Enforcement Count
* Status (Active / Archived)
* View Details

Each row should represent **one ordinance implemented in a barangay**.

---

### 3. Violation Rate Per Year

Add a **violation statistics panel** that calculates:

* Total number of people who violated each ordinance
* Annual violation rate
* Violations per barangay per year

Display the values as **numeric indicators instead of graphs**.

Example:

Anti-Littering Ordinance
Violations 2023: 120
Violations 2024: 185
Increase Rate: +54%

Curfew Ordinance
Violations 2023: 90
Violations 2024: 60
Decrease Rate: −33%

---

### 4. Ordinance Details Function

When clicking **View Details**, open a page showing:

* Full ordinance description
* Barangay implementing the ordinance
* Total number of violations recorded
* Yearly violation statistics
* Enforcement history
* Blockchain transaction verification

Show blockchain fields:

* Transaction Hash (TxHash)
* Block Number
* Timestamp
* Recorded By
* Blockchain Status

Include a **“Verify Record on Blockchain” button**.

---

### 5. Real-Time Record Behavior

Ensure the dashboard automatically updates when:

* A new ordinance is added
* A violation record is logged
* A blockchain verification occurs

The **Published Documents, Verified on Chain, and Active Ordinances counters must update automatically**.

---

### 6. Dashboard Design

Design the dashboard to be **clean, professional, and easy to use**.

Design requirements:

* Minimal and organized layout
* Clear data cards for counters
* Simple ordinance table layout
* Soft shadows and spacing
* Easy navigation from sidebar
* Responsive spacing for large data tables

Focus on **clarity and usability for government administrators**.

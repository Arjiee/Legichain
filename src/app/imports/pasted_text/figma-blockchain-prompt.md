# Figma AI Prompt – Improve Blockchain Section in Admin Panel

**Prompt:**

Update the **Blockchain section in the Admin Panel** of the **GMA Cavite Blockchain Barangay Ordinance Monitoring System prototype** to make it **more detailed, useful, and informative for administrators**. The blockchain module should clearly display **verified ordinance and violation records stored on the blockchain**.

---

# 1. Improve Blockchain Records Table

Update the **Blockchain Records table** to display detailed blockchain transaction information.

The table must contain the following columns:

* Transaction Hash (TxHash)
* Block Number
* Ordinance ID
* Barangay
* Record Type (Ordinance / Violation Case)
* Action Recorded (Created, Updated, Verified)
* Timestamp
* Recorded By
* Verification Status
* View Details

This table should show **all blockchain transactions related to ordinance records and violation cases**.

---

# 2. Blockchain Transaction Details Page

When the admin clicks **View Details**, open a **Blockchain Transaction Details page** that displays complete blockchain information:

* Transaction Hash
* Block Number
* Previous Block Hash
* Smart Contract Address
* Ordinance ID
* Barangay Name
* Violation Case ID (if applicable)
* Action Recorded
* Timestamp
* Recorded By
* Verification Status

Include a **Blockchain Verification Badge** showing if the transaction is verified.

---

# 3. Blockchain Statistics Panel

Add a **Blockchain Statistics section** above the records table showing:

* Total Blockchain Transactions
* Total Verified Records
* Total Ordinance Records Stored
* Total Violation Records Stored
* Latest Block Number

These statistics should **update automatically when new records are added**.

---

# 4. Link Blockchain to Ordinance Records

Ensure that every time an **ordinance or violation record is created or updated**, the system automatically generates a **blockchain transaction entry**.

Example actions recorded on blockchain:

* New ordinance added
* Violation case recorded
* Violation record updated
* Ordinance verified on chain

These actions must appear in both:

* **Blockchain Records**
* **Audit Logs**

---

# 5. Add Search and Filters

Allow administrators to filter blockchain records by:

* Barangay
* Ordinance ID
* Record Type
* Date Range
* Verification Status

Include a **search bar** to quickly locate transactions using **TxHash or Ordinance ID**.

---

# 6. Design Improvements

The updated blockchain page should:

* look like a **blockchain explorer interface**
* use a **clean professional layout**
* highlight **TxHash and Block Numbers clearly**
* maintain consistent design with the rest of the Admin Dashboard

The **Blockchain section in the Admin Panel will become a detailed blockchain explorer for the system**, allowing administrators to **track ordinance records, violation cases, and verification transactions securely and transparently** within the prototype.
